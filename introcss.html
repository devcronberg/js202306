    Introduktion til CSS

    Hvad er CSS

    CSS (Cascading Style Sheets) er et opmærkningssprog som fortæller hvordan browseren skal vise html/xml. Hvor indhold er defineret i HTML er design altså defineret i CSS.

    User Agent Stylesheet

    Når du skriver HTML og viser det “råt” i en browser er der allerede design (overskrift, fonte, margin mv), men det er fordi browseren har et standardiseret og predefineret udgangspunkt kaldet user agent stylesheet. Når du selv skrive CSS så overskriver eller tilføjer til dette styelsheet.

    Elementer i CSS

    Helt grundlæggende består CSS af en samling elementer som er opdelt i to ting

    • en søgedefinition (selector) som fortæller browseren hvilke DOM elementer som skal påvirkes
    • egenskaber (properties) som fortæller browseren hvordan de skal påvirkes

    Syntaks

    En selector skal efterfølges af tuborgklammer, og de enkelte egenskaber skal afsluttes med semikolon.

    Overordnet er browseren ligeglad med tabulering, mellemrum og linjeskift - det skal kun være for udviklernes skyld. Store CSS filer kan man derfor vælge at minificere for at gøre dem så små som muligt.

    Knyttes til HTML

    CSS skal loades af browseren og defineret i den HTML det skal påvirke. Dette kan blandt andet ske

    • ekstern
    • i head
    • inline

    W3C

    CSS som oprindeligt er skabt af blandt andet Håkon Vium Lie og Tim Bernes-Lee, blev i 1994 (28 år siden) standardiseret hos W3C.

    Fra version 3 er CSS splittet og frigivet i moduler - se https://www.w3.org/Style/CSS/current-work. Her skal man især lægge mærke til de forskellige niveauer hos W3C:

    Niveau Beskrivelse
    WD Working draft
    CR Candidate recommendation
    PR proposed recommendation
    REC Recommendation

    Mange af de nævnte moduler er ikke REC hvilket betyder, at de ikke nødvendigvis virker i alle browsere. En god side til at være sikker er https://caniuse.com, men det bedste er naturligvis test - evt. gennem https://www.browserstack.com/.

    W3C varetager ligeledes standardisering for mange andre web relaterede teknologier. Herudover stiller W3C forskellige værktøjer til rådighed - herunder især validering af HTML og CSS.

    CSS Selectors

    En CSS selector er en søge streng som browseren benytter til at finde de elementer der skal påvirkes af CSS.

    Det kunne eksempelvis være alle p-opmærkninger:

    p {
}

    Alle h1’ere under en section:

    section h1 {
}

    Alle elementer med klassen rød:

    .rød {
}

    Elementet med ‘knap1’ som id:

    #knap1 {
}

    Der findes et hav af andre selectors og nogle kan være meget avancerede.

    CSS properties

    Når først elementer er fundet med en CSS selector kan de påvirkes med en masse forskellige egenskaber relateret til eksempelvis

    • Farver
    • Mønstre
    • Fonte
    • Størrelser
    • Placeringer
    • Rammer
    • Animationer
    • Dynamiske egenskaber relateret til klient/størrelse
    • Meget mere

    Eksempler på CSS

    Her er et par små eksempler - men der er rigtig mange egenskaber.

    Tekster

    p {
    font-size: 2em;
    color: red;
    background-color: white;
    font-style: italic;
    width: 200px;
}

    CSS Frameworks

    I moderne websites vælger mange at udnytte open source baserede CSS framework for på den måde at få et nemt og effektiv udgangspunkt til design af knapper, lister, grids, sektioner og meget mere.

    De meste populære frameworks er

    • Bootstrap
    • Semantic UI / Fomantic UI
    • Materialize
    • Bulma
    • UiKit

    men der er mange flere og det er lidt en religion.

    Prøv eventuelt at lege med Semantic med udgangspunkt i følgende HTML/CSS og deres dokumentation til knapper og Rating-modulet:

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <link
      rel="stylesheet"
      href="https://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.4.1/semantic.min.css"
    />
    <style>
      body {
        margin: 20px;
      }
    </style>
    <title>Min demo</title>
  </head>
  <body>
    <div class="ui container">
      <h1 class="ui header">Min demo</h1>

      <div class="ui animated fade huge green button" tabindex="0">
        <div class="visible content">Se min</div>
        <div class="hidden content">knap</div>
      </div>

      <div style="margin-top: 20px">
        <div class="ui star rating" data-rating="3"></div>
      </div>
    </div>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.4.1/semantic.min.js"></script>
    <script>
      $(".rating").rating("enable");
    </script>
  </body>
</html>

    Utility-first CSS framework

    Det er de sidste par år blevet populært at benytte utility-first CSS framework som eksempelvis TailWind. Det er mere rå frameworks som kræver lidt mere viden, men til gengæld er man ikke hårdt bundet til et givet design. I TailWind eksmepelvis arbejder med med predefinerede klasser som direkte matcher CSS-egenskaber.

    Her er et kort eksempel:

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/tailwindcss/2.0.2/tailwind.min.css" />
  </head>
  <body>
    <div class="container">
      <h1 class="p-5 text-5xl text-red-700 animate-pulse">Min side</h1>
      <p class="m-5 p-5 rounded border shadow-xl border-black bg-gray-200">Lorem, ipsum dolor sit amet consectetur adipisicing elit. Ea, error nesciunt deserunt consequuntur, nisi, unde blanditiis aut amet deleniti odio vero dicta corrupti! Nihil minus porro inventore magnam sunt temporibus.</p>
    </div>
  </body>
</html>

    Typisk vil man ikke benytte TailWind fra en CDN men gennem en Node-udviklingsproces. Se evt denne gode serie om TailWind.

    CSS pre- og postprocessor

    Preprocessorer

    CSS er som udgangspunkt ikke et programmeringssprog og i store CSS frameworks kan det være en fordel at kunne skrive CSS på en mere “programmeringsmæssig måde”. Derfor findes der flere CSS preprocessorer som i virkeligheden skal ses som en transpiler der oversætter en syntaks til CSS. Der er især to meget benyttede kaldet SASS og LESS.

    Eksempelvis er dette LESS kode:

    @width: 10px;
@height: @width + 10px;

#header {
    width: @width;
    height: @height;
}

    og det minder syntaktmæssigt meget om CSS. Men der er tilføjet variabler (@) og beregninger (+) så en transpilering vil skabe følgende CSS:

     #header {
  width: 10px;
  height: 20px;
}

    Både LESS og SASS tilføjer en masse funktionalitet men efterhånden som CSS i W3C’s standard bliver mere og mere moden, vil behovet for preprocessorer bliver mindre og mindre.

    Postprocessorer

    Hvor LESS og SASS er en del af selve udviklingsprocessen, og hjælper med at skabe CSS under udvikling, findes der også postprocessorer som afvikles i forbindelse med distribution. Det kunne eksempelvis være

    • Kontrol af CSS mod W3C
    • Minificering (fjerner mellemrum og linjeskift)
    • Transpilering af helt ny CSS tilbage til ældre CSS for support

    Se eksempelvis PostCSS eller CSSNext.
