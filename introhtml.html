    Introduktion til HTML

    Hvad er HTML

    HTML (HyperText Markup Language) er et relativt simpelt opmærkningssprog, som har til formål at beskrive data/indhold i en webapplikation.

    Ved hjælp af opmærkninger som eksempelvis

    <!-- paragraf -->
<p>test</p>

<!-- link -->
<p>Tekst med <a href="...">link</a></p>

<!-- billede -->
<img src="..." />

    kan der blandt andet beskrives

    • tekst
    • links
    • billeder
    • formularer

    HTML beskriver ikke på nogen måde design, som varetages af CSS (Cascading Style Sheets), eller dynamik, som varetages af ES (JavaScript) samt WASM (WebAssembly).

    Opmærkninger

    Opmærkninger kan enten bestå af data som eksempelvis

    <h1>Overskrift</h1>
<p>tekst</p>

    eller uden data

    <hr />

    og opmærkningerne placeres i et træ (opmærkninger i opmærkning i opmærkninger i …)

    <section>
    <div>
        <p>tekst</p>
        <div>
            <p>
                tekst
                <span>tekst</span>
            </p>
        </div>
    </div>
</section>

    Alle opmærkninger kan have attributter bestående af en nøgle og en værdi

    <div id="minDiv">tekst</div>
<p id="id1" class="class1 class2" style="color: red">tekst</p>
<img src="billeder/billede1.png" alt="Mit billede" />

    Mellemrum har ingen betydning

    Hverken mellemrum eller tabuleringer har nogen betydning i HTML - de gør det blot nemmere at læse for “os”.

    Således tolkes følgende

    <p>Tekst Tekst</p>
<div>
    <p>Ny tekst</p>
</div>

    som

    <p>Tekst Tekst</p>
<div><p>Ny tekst</p></div>

    af browseren. Det er udelukkende et enkelt mellemrum i tekster som har betydning.

    W3C

    HTML (HyperText Markup Language), som oprindeligt er skabt af blandt andet Tim Bernes-Lee insireret af ældre teknologier som HyperText (Ted Nelson), blev i 1994 (28 år siden) standardiseret hos W3C. Herfra er der efterfølgende frigivet mange versioner, og i skrivende stund (2020) er den seneste version 5.2.

    W3C varetager ligeledes standardisering for mange andre web relaterede teknologier. Herudover stiller W3C forskellige værktøjer til rådighed - herunder især validering af HTML og CSS.

    Semantik

    Du skal se opmærkninger som måden at beskrive/definere data (tekst, afsnit, billeder, links mv). Nogle udviklere/designere vælger at være meget explicit i brugen af opmærkninger og benytter hele paletten af HTML. De er meget semanantiske i deres tilgang til opmærkningerne. Andre er mere orienteret mod CSS (styling) og går ikke så meget op i de opmærkningerne som ikke har konkret funktionalitet i browsere (som links og billeder elsempelvis).

    Eksempelvis kan en overskrift (h1), tekst (p) og en tabel (table, tr, td) skrives meget semantisk (HTML “mindset”):

    <!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Test</title>
    </head>
    <body>
        <section>
            <h1>Overskrift</h1>

            <p>Paragraf med tekst</p>

            <table width="50%">
                <tr>
                    <td>Række 1 Kolonne 1</td>
                    <td>Række 1 Kolonne 2</td>
                </tr>
                <tr>
                    <td>Række 2 Kolonne 1</td>
                    <td>Række 2 Kolonne 2</td>
                </tr>
            </table>
        </section>
    </body>
</html>

    Browseren vil benytte det indbyggede stylesheet (definering af stil) kaldet “user agent stylesheet” til at vise opmærkningerne.

    Men samme side kan også skrives med et CSS “mindset”:

    <!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Test</title>
        <link
            rel="stylesheet"
            href="https://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.4.1/semantic.min.css"
        />
    </head>
    <body>
        <div>
            <div class="ui header">Overskrift</div>

            <div class="ui segment basic">Paragraf med tekst</div>

            <div class="ui grid" style="width: 50%">
                <div class="ui row">
                    <div class="ui eight wide column">Række 1 Kolonne 1</div>
                    <div class="ui eight wide column">Række 1 Kolonne 2</div>
                </div>
                <div class="ui row">
                    <div class="ui eight wide column">Række 2 Kolonne 1</div>
                    <div class="ui eight wide column">Række 2 Kolonne 2</div>
                </div>
            </div>
        </div>
    </body>
</html>

    Her benyttes et CSS framework kaldet SemanticUI, som har defineret de forskellige klasse navne, men det kan være hvad som helst.

    Der er som udgangspunkt ingen forskel for browseren på brugen af semantiske opmærkninger som h1 og p tilgang i forhold til brugen af div - det er op til udvikleren. Tit er de to også kombineret - eksempelvis

    <h1 class="ui header">Overskrift</h1>

    Her er h1 benyttet sammen med css klasser og kunne ligeså godt være skrevet som

    <div class="ui header">Overskrift</div>

    eller

    <p class="ui header">Overskrift</p>

    De fleste vælger at kombinere de gamle Tim Berners-Lee opmærkninger med css klasser - altså

    <h1 class="ui header">Overskrift</h1>

    men det har ingen praktisk betydning relateret til browseren.

    Skabelon

    En HTML side følger en grundlæggende skabelon med en html opmærkning samt en head og en body:

    <!DOCTYPE html>
<html>
    <head>
        <!-- ... -->
    </head>
    <body>
        <!-- ... -->
    </body>
</html>

    Doctype

    Doctypen beskriver hvilken version af HTML browseren kan forvente.

    Tidligere var den meget specifik med henvisning til skemaer, men i dag er den mere generel.

    <!DOCTYPE html>

    Ovennævnte betyder blot, at siden er en HTML5+ side.

    Header

    Beskriver type metadata

    • meta
    • css
    • javascript
    <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta name="description" content="Min side der består af ...." />
    <meta name="keywords" content="HTML, CSS, JavaScript" />
    <meta name="author" content="Michell Cronberg" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>Min side</title>
</head>

    Nogle meta data er SEO relateret (Search Engine Optimization) medens andre er relateret til browserens forståelse af siden.

    Body

    Består af opmærkningerne til siden som renderes af browseren.

    Grundlæggende eksempel på en skabelon

    <!DOCTYPE html>
<html lang="da">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <meta http-equiv="X-UA-Compatible" content="ie=edge" />
        <title>Min side</title>
    </head>

    <body></body>
</html>

    Block og Inline

    Helt grundlæggende kan opmærkninger være enten block elementer som af browseren placeres under hinanden, og inline elementer som af browseren placeres ved siden af hinanden. Eksempelvis er p-opmærkningen et block element

    <p>tekst</p>
<p>tekst</p>
<p>tekst</p>

    Ovennævnte vil renderer tre afsnit under hinanden.

    Følgende er eksempler på inline elementer

    <img src="https://cdn.cronberg.dk/kurser/div/csbog.jpg" width="200" alt="bog" />
<img src="https://cdn.cronberg.dk/kurser/div/csbog.jpg" width="200" alt="bog" />
<img src="https://cdn.cronberg.dk/kurser/div/csbog.jpg" width="200" alt="bog" />

    Det vil renderer tre billeder ved siden af hinanden.

    Men husk at det kan ændres gennem css.

    Eksempler på HTML

    Det ligger uden for dette modul at beskrive opmærkningerne i detaljer, men her er et eksempel på en HTML-side med forskellige opmærkninger:

    <!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Min side</title>
    </head>
    <body>
        <h1>Overskrift</h1>
        <p>Paragraf med tekst</p>

        <!-- Liste -->
        <ul>
            <li>Element 1</li>
            <li>Element 2</li>
            <li>Element 3</li>
        </ul>

        <!-- Tabel -->
        <table>
            <tr>
                <td>række 1 kolonne 1</td>
                <td>række 1 kolonne 2</td>
            </tr>
            <tr>
                <td>række 2 kolonne 1</td>
                <td>række 2 kolonne 2</td>
            </tr>
        </table>

        <!-- Formular -->
        <form><input type="text" /><button>Klik på mig</button></form>
    </body>
</html>

    Web komponenter

    Du vil muligvis falde over opmærkninger som ikke ligner noget der kommer fra W3C - eksempelvis

    <p>Demo af terning</p>
<cronberg-dice></cronberg-dice>

    hvor “cronberg-dice” tydeligvis ikke er en W3C standard. Det kaldes en web komponent (Web Component) og giver mulighed for at skabe egne komponenter med indkapslet logik og design som dermed er nemme at genbruge.

    Du kan se et eksempel på en web komponent på devcronberg/WebComponentDemo hvor ovennævnte terning er brugt, eller hente js-filen direkte som

    <script src="https://devcronberg.github.io/WebComponentDemo/dice.js"></script>

    og prøve terningen som

    <cronberg-dice cheat="false" debug="false" value="1"></cronberg-dice>

    Biblioteker

    Der findes masser af biblioteker med web components på nettet - eksempelvis webcomponents.org eller Googles Polymer Project

    HTML pre- og postprocessor

    Preprocessorer

    Du vil muligvis falde over begreber som

    • markdown
    • pug
    • haml
    • slim

    Det er såkaldte HTML preprocessorer som typisk er en definitation af en mere simpel syntaks end ren html. Til gengæld kræver det en konvertering (transpilering) af den simple syntaks til html inden det kan bruges - for html er det eneste browseren forstår.

    Eksempelvis kan følgende markdown

    # Overskrift Dette er et afsnit - værdi 1 - værdi 2 - værdi 3

    konverteres til følgende html

    <h1>Overskrift</h1>
<p>Dette er et afsnit</p>
<ul>
    <li>værdi 1</li>
    <li>værdi 2</li>
    <li>værdi 3</li>
</ul>

    Så det er nemmere at skrive markdown (md) end HTML- men kræver altså et ekstra step for at generere HTML.

    Postprocessorer

    Når først du har skrevet din HTML findes der scripts/moduler/applikationer - såkaldte postprocessorer - som kan gøre HTML klar til drift. Det kunne eksempelvis være

    • kontrol af HTML mod W3C (linter)
    • kontrol af links og billeder
    • minificering af HTML (fjerner mellemrum og linjeskift mv)

    Disse scripts er typisk en del af udviklingsprocessen i en eller anden form - måske afviklet i forbindelse med build.
