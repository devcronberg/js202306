    Introduktion til JavaScript

    JavaScript er det programmeringssprog der benyttes i alle moderne browsere, og er i dag den eneste mulighed for at skabe en dynamisk side. Der findes også WebAssembly (WASM) men det benyttes til at skabe komponenter som kan kaldes fra JavaScript.

    Sproget er det populære programmeringssprog (developer survey) fordi det er effektivt og “simpelt”. Samtidigt er det behæftet med forskellige fejl/uhensigtsmæssigheder fra fødsel - som dog er rettet i de senere versioner (så godt som det nu er muligt).

    Sproget benyttes i forskellige klienter og runtimes - mest kendt er NodeJS som benyttes Google’s V8 motor til at kompilere og afvikle JavaScript på desktop/server.

    Oracle har varemærket på JavaScript og er i princippet de eneste som må kalde sproget for JavaScript. Derfor findes der forskellige kreative navne på implementationer hos firmaer som blandt andet udvikler browsere - herunder

    • Carakan (Opera)
    • SpiderMonkey (Firefox)
    • V8 (Google)
    • JScript (Microsoft)
    • og mange flere

    Sproget er skabt af Brendan Eich (Netscape) i 1995 inspireret af sprog

    • C
      • syntaks
    • Scheme/Lisp
      • funktionsorienteret
    • Self/Smalltalk
      • objektorienteret

    Sproget er typesvagt samt dynamisk og har features relateret til objektorienteret programmering (baseret på prototype) og funktionsorienteret programmering.

    ECMA Script

    Sproget er standardiseret af ECMA (tidligere European Computer Manufacturers Association - nu Ecma International) første gang i 1997 (@cs DateTime.Now.Year - 1997 @ år siden) - og kaldes derfor typisk ES (ECMAScript).

    Der er frigivet en del versioner gennem årene:

    • 1997 ES v1
    • 1997 ES v2
    • 1999 ES v3
    • 2009 ES v5
    • 2015 ES v6
    • 2017 ES 2017
    • 2018 ES 2018
    • 2019 ES 2019
    • 2020 ES 2020
    • 2021 ES 2021

    De versioner man typisk hører mest om er ES3 (stor opdatering) og især ES6 (stor opdatering).

    Platform

    Man kan vælge at afvikle JavaScript filer direkte i browseren, men ved hjælp af NodeJS kan JavaScript også benyttes på desktop eller server til blandt andet automatisering eller applikationsafvikling.

    Introduktion til JavaScript

    Der er masser ressourcer på nettet relateret til programmering med JavaScript (ECMAScript), men her er et par ting som du bør vide:

    • Det kan (som det eneste scriptsprog) afvikles i alle browsere
    • Det kan afvikles gennem Node.js
    • Det har en C syntaks
    • Det er typefrit
    • Det er dynamisk
    • Det er objektorienteret

    Eksempler

    Her er et par korte eksempler med kommentarer:

    // Bør altid være i toppen af scope i alt JavaScript
"use strict";

// Udskriv på consol (node eller browser)
console.log("Test");

// erklæring og tildeling af variabler
let a = 1;
let b = "Teknologisk"; // eller 'Teknologisk'

// beregninger
a++;
a = a + 10;
console.log("a = " + a);

// betingelser
if (a === 12) {
    console.log("a = 12");
}

if (a === 12 && b !== "JS") {
    console.log("a = 12 og b er forskellig fra 'JS'");
}

// løkke - tæller 0 - 4
for (var i = 0; i < 5; i++) {
    console.log(i);
}

// funktion
function add(a, b) {
    return a + b;
}
let resultat = add(10, 20);
console.log("10 + 20 =" + resultat);

// Objekt
let c = {
    navn: "Mathias",
    alder: 14,
};
console.log(c);

// Klasse
class Person {
    constructor(navn, alder) {
        this.navn = navn;
        this.alder = alder;
    }
    skriv() {
        console.log(`Jeg hedder ${this.navn} og er ${this.alder} gammel.`);
    }
}

let p = new Person("Mikkel", 17);
p.skriv();

    Moduler

    I det oprindelige ECMA Script kan man udelukkende indkapsle data og funktionalitet i funktioner, og det kan gøre en større kodebase meget kompleks at udvikle. Derfor har man søgt efter muligheder for at modulalisere ES, og flere mønstre/formater/standarder er blevet skabt.

    • Closure og funktioner i “ren” ES
    • CommonJS
    • AMD (Asynchronous Module Definition)
    • UMD (Universal Module Definition)

    De sidste tre kræver en transpiler som WebPack, Browserify eller Parcel for at virke i en browser, og Node har valgt at benytte CommonJS som standard.

    ES6 moduler

    Inden for de sidste par år er ES6 moduler blevet en mulighed i alle browsere og Node så det er sikkert den vej moduler i fremtiden vil blive udviklet.

    I ES6 er en .js fil et modul der som helt af sig selv er en container hvor alt er privat, og hvis man ønsker at andre skal have adgang til data og funktionalitet skal det eksporteres.

    AJAX

    AJAX er et akronym for Async JavaScript And XML og dækker over muligheden for at benytte det i alle browsere indbyggede XML HTTP Request Object til HTTP kommunikation. Objektet er tilgængeligt direkte men de fleste benytter et bibliotek som jQuery (som har AJAX metoder) eller Axios.

    Eksempel med Axios

    Her er et eksempel med brug af Axios i en browser:

    <!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Kommuner</title>
        <h1>Alle kommuner i Danmark</h1>
    </head>
    <body>
        <ul id="kommuner"></ul>
        <script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>
        <script>
            hentData();

            async function hentData() {
                const response = await axios.get(
                    "https://dawa.aws.dk/kommuner/"
                );
                const ol = document.getElementById("kommuner");
                for (var i = 0; i < response.data.length; i++) {
                    const li = document.createElement("li");
                    li.innerHTML = response.data[i].navn;
                    ol.appendChild(li);
                }
            }
        </script>
    </body>
</html>

    Bemærk, at biblioteket hentes først i jsdelivr (men det findes mange steder - eller kan hentes lokalt).

    Her er samme funktion afviklet via node, men det kræver at Axios er hentet med

    npm i axios --save

    Herefter kan følgende hente kommuner og udskrive på konsol:

    const axios = require("axios").default;

hentData();

async function hentData() {
    const response = await axios.get("https://dawa.aws.dk/kommuner/");
    for (var i = 0; i < response.data.length; i++) {
        console.log(response.data[i].navn);
    }
}

    Polyfill

    Man skal være lidt opmærksom på hvilke version af EcmaScript man benytter fordi der vil opstå fejl i ældre browsere hvis den pågældende version ikke understøttes. Se eksempelvis classList og String.ReplaceAll.

    Problemet kan løses med et polyfill som eksempelvis classList eller følgende kode til ReplaceAll:

    if (!String.prototype.replaceAll) {
    String.prototype.replaceAll = function (str, newStr) {
        // If a regex pattern
        if (
            Object.prototype.toString.call(str).toLowerCase() ===
            "[object regexp]"
        ) {
            return this.replace(str, newStr);
        }

        // If a string
        return this.replace(new RegExp(str, "g"), newStr);
    };
}

    Der findes en del biblioteker som kan hjælpe - eksempelvis https://polyfill.io/v3/

    Lintere

    En af de største udfordringer ved udvikling af JavaScript er at sproget er typefrit og dynamisk. Det kan give en masse mulighed i udvikling af logik, men det betyder også at udviklingsmiljøer har svært ved at tilbyde hjælp. Yderligere er der en masse “best practice” i JavaScript som kan være svær at overholde - selv for erfarne udviklere.

    Derfor findes der flere produktioner - kaldet lintere - til JavaScript, som har til formål at finde fejl og komme med advarsler ud fra et givet sæt af regler.

    ESLint

    En af de mere populære er ESLint som kan tilrettes på alle mulige måder. Eksempelvis kan JavaScript kode som

    let a = b;

    være et problem fordi b ikke er erklæret inden brug, og a i øvrigt ikke er benyttet. Begge dele er vigtig information men ikke i sig selv en fejl i JavaScript.

    Derfor vil du til finde ESLint, JSLint eller andre lignende linter være en del af et JavaScript projekt.

    Transpilere

    JavaScript kan være en udfordring grundet både designfejl, manglende typesikkerhed samt krav om kompatibilitet med ældre versioner af runtime i browsere. Derfor har der været tradition for, at kode blev skrevet i andre sprog (eksempelvis Java eller Python) og så transpileret til JavaScript.

    TypeScript (TS)

    De senere år er der dog dukket andre typer af transpilere op som man typisk ser i et større JavaScript projekt. Microsofts TypeScript er en af de populære transpilere. Den tilføjer både typer, nye features til sproget, retter gamle fejl og transpilere ny og moderne JavaScript til ældre versioner.

    TypeScript er et supersæt af JavaScript forstået således, at en JS fil helt automatisk er en TS fil men der kan tilføjes features til TS som så transileres til JS.

    Eksempelvis er følgende kode TS og ikke JS:

    function lægSammen(a: number, b: number): number {
    return a + b;
}

let a: number = lægSammen(1, 1);

    Læg mærke til definitionen af typer som JS ikke forstår, men som giver en masse fordele i forbindelse med udvikling af kode. TS koden bliver transpileret til JS som følger:

    function lægSammen(a, b) {
    return a + b;
}
let a = lægSammen(1, 1);

    Nu er typedefinitionen væk og koden kan forstås af JS runtimes.

    Et andet eksempel er følgende:

    namespace Cronberg {
    class Person {}

    class Elev extends Person {}
}

    hvilket er logisk objektorienteret kode, men hvis det skal virke i ældre browsere som benytter ES5 blivr det transpileret til

    var __extends =
    (this && this.__extends) ||
    (function () {
        var extendStatics = function (d, b) {
            extendStatics =
                Object.setPrototypeOf ||
                ({ __proto__: [] } instanceof Array &&
                    function (d, b) {
                        d.__proto__ = b;
                    }) ||
                function (d, b) {
                    for (var p in b)
                        if (Object.prototype.hasOwnProperty.call(b, p))
                            d[p] = b[p];
                };
            return extendStatics(d, b);
        };
        return function (d, b) {
            extendStatics(d, b);
            function __() {
                this.constructor = d;
            }
            d.prototype =
                b === null
                    ? Object.create(b)
                    : ((__.prototype = b.prototype), new __());
        };
    })();
var Cronberg;
(function (Cronberg) {
    var Person = /** @class */ (function () {
        function Person() {}
        return Person;
    })();
    var Elev = /** @class */ (function (_super) {
        __extends(Elev, _super);
        function Elev() {
            return (_super !== null && _super.apply(this, arguments)) || this;
        }
        return Elev;
    })(Person);
})(Cronberg || (Cronberg = {}));

    Babel

    En anden populær transpiler er Babel som har lignende funktionalitet men som ikke er typestærk men derimod stærk i transpilering af helt nye features i JS til ældre versioner.
