    Introduktion til VS Code

    Visual Studio Code

    Visual Studio Code (forkortet VSC) er på få år blevet en af de mest benyttede teksteditorer til alle former for udvikling. I Stack Overflow’s undersøgelse fra 2019 havde den overgået både Visual Studio, Notepad++ og VIM.

    Den kan installeres på alle platforme fra

    https://code.visualstudio.com/

    og er skabt til udvikling med indbygget funktionalitet relateret til

    • editering
    • extensions
    • git support
    • brugertilpasning

    samtidigt er den super hurtig.

    Den er bygget i HTML, CSS og JavaScript på Electron platformen af Microsoft.

    Video: Installering af VSC og extensions

    Her er en kort video som viser installation af VSC og en enkelt extension.

    Grundlæggende settings

    På kan overvejer at sætte VS op med

    • Automatisk omdøbning af opmærkninger
      • Linked editing under settings
    • Åbn tom editor hver gang (uden tidligere filer)
      • Restore windows under settings

    Dokumentation

    Du bør bruge noget tid på at læse dokumentationen fra deres website - det er den bedste måde at lære om de forskellige features. Det vil spare mange timers udvikling. Læs (skim) blandt andet om

    • Editering
    • Extensions
    • IntelliSense
    • Code Navigation
    • Refactoring
    • Debugging
    • Version Control
    • Integrated Terminal
    • Tasks
    • Snippets

    Brug af VSC til HTML udvikling

    Hvis du benytter VSC til HTML udvikling kan du optimere brugen på mange måder.

    Emmet

    Den største feature er uden tvivl Emmet som du kan se som dynamiske snippets. Hvis du eksempelvis indtaster

    h1

    og trykker tabulering bliver det til

    <h1></h1>

    og sætter cursoren det rigtige sted.

    Skriver du

    div>span

    bliver det til

    <div><span></span></div>

    Emmet benytter kendte templates fra CSS hvor eksempelvis # betyder id og . betyder klasse. Således vil dette

    div#d1>span.c1

    blive til

    <div id="d1"><span class="c1"></span></div>

    Wrap

    Du kan også omkranse med opmærkninger ved at bruge f1-menuen (Windows) og skrive wrap og vælge “Wrap with appreviation” og så skrive en emmet snippet. På den måde kan

    link

    hurtigt blive til

    <a href="" id="a1">link</a>

    ved at skrive “a#a1” i “Wrap with appreviation”.

    Actions

    På f1-menuen kan de mange “actions” findes ved at skrive “emmet” - prøv eksempelvis “Update image size” som automatisk sætte width og height.

    Egne snippets

    Du kan også oprette dine egne emmet snippets i VSC. Under settings skal du angive en mappe med snippets.

    Dokumentation

    Det er kun toppen af isbjerget og du bør bruge lidt tid på at lege med de mange muligheder. Se https://docs.emmet.io/cheat-sheet.

    Eksempler på Emmet

    Her er et par af mine mest benyttede snippets:

    h1
<h1></h1>

<p>Lorem ....</p>

<nav>
    <ul>
        <li></li>
    </ul>
</nav>

<p></p>
<blockquote></blockquote>

ul>li*5
<ul>
    <li></li>
    <li></li>
    <li></li>
    <li></li>
    <li></li>
</ul>

table>tr*3>td*3
<table>
  <tr>
    <td></td>
    <td></td>
    <td></td>
  </tr>
  <tr>
    <td></td>
    <td></td>
    <td></td>
  </tr>
  <tr>
    <td></td>
    <td></td>
    <td></td>
  </tr>
</table>

ul>li.item$*5
<ul>
    <li class="item1"></li>
    <li class="item2"></li>
    <li class="item3"></li>
    <li class="item4"></li>
    <li class="item5"></li>
</ul>

h$[title=item$]{Header $}*3

<h2 title="item2">Header 2</h2>
<h3 title="item3">Header 3</h3>

ul>li.item$$$*5
<ul>
    <li class="item001"></li>
    <li class="item002"></li>
    <li class="item003"></li>
    <li class="item004"></li>
    <li class="item005"></li>
</ul>

div#header

<form id="search" class="wide"></form>

p.class1.class2.class3
<p class="class1 class2 class3"></p>

p[title="Hello world"]
<p title="Hello world"></p>

td[rowspan=2 colspan=3 title]
<td rowspan="2" colspan="3" title=""></td>

a{Click me}

p>{Click }+a{here}+{ to continue}

a:link
<a href="http://"></a>

link:css
<link rel="stylesheet" href="style.css" />

    Extensions

    Der findes mange extensions relateret til HTML - her er nogle af dem jeg benytter

    • Live Server (ritwickdey.liveserver)
      • Indbygget web server
    • Prettier (esbenp.prettier-vscode)
      • Formatering af blandt andet HTML
      • Sæt evt “Format On Save” så der formateret når du gemmer filer
    • HTMLHint (mkaufman.htmlhint)
      • Kontrol af opmærkning mod et sæt regler (ikke W3C)
    • Encode Decode (mitchdenny.ecdc)
      • encoding af eksempelvis ø til ø
    • W3C Web Validator
      • Validering mod W3C

    Snippets

    Husk du kan nemt skabe dine egne snippets hvis der er nogle opmærkninger du bruger meget.

    Brug af VSC til CSS udvikling

    Hvis du benytter VSC til CSS udvikling kan du optimere brugen på mange måder.

    Emmet

    Der findes en masse snippets relateret til CSS. Eksempelvis vil

    bg

    blive til

    background: #000;

    eller

    m20

    til

    margin: 20px;

    Se de mangle muligheder på https://docs.emmet.io/cheat-sheet

    Extensions

    Der findes mange extensions relateret til CSS - herunder er nogle af dem jeg selv bruger

    • Prettier (esbenp.prettier-vscode)
      • Formatering af CSS mv
    • HTML CSS Support (ecmel.vscode-html-css)
      • Hjælp til CSS id/klasse i opmærkninger
    • cdnjs (JakeWilson.vscode-cdnjs)
      • Nem tilgang til css/script fra cndjs

    Brug af VSC til JS/TS udvikling

    Hvis du benytter VSC til JavaScript og TypeScript udvikling kan du optimere brugen på mange måder. Start med at læse https://code.visualstudio.com/docs/languages/javascript for at se alle mulighederne.

    Snippets

    Der findes et hav af snippets i VSC til .js og .ts filer, og det er nemt at skabe sine egne. Her er et par af de mest benyttede:

    for (for løkke)
if (betingelse)
ifelse (betingelse)
cw (console.log)

    Extensions

    Jeg har følgende extensions installeret

    • Live Server (ritwickdey.liveserver)
      • Indbygget web server
    • Visual Studio IntelliCode (visualstudioexptteam.vscodeintellicode)
      • Udvidet intellicense
    • Prettier (esbenp.prettier-vscode)
      • Formatering af blandt andet HTML
      • Sæt evt “Format On Save” så der formateret når du gemmer filer
    • ESLint (dbaeumer.vscode-eslint)
      • Linter
    • TSLint (ms-vscode.vscode-typescript-tslint-plugin)
      • Linter
    • Debugger for Chrome (msjsdiag.debugger-for-chrome)
    • Vetur (octref.vetur)
      • Vue udvikling (men der findes også til Angular, React mv)
    • cdnjs (JakeWilson.vscode-cdnjs)
      • Nem tilgang til css/script fra cndjs
    • Bracket Pair Colorizer 2 (CoenraadS.bracket-pair-colorizer-2)
      • Farve på tuborgklammer
    • indent-rainbow (oderwat.indent-rainbow)
      • Visuel indrykning

    Debugging

    Debuggeren i VSC er helt forrygende og du skal læse lidt om den på https://code.visualstudio.com/docs/languages/javascript#_debugging

    Brug af VSC til C# udvikling

    Hvis du arbejder med VSC til C# udvikling er det især snippets, extensions, og debuggeren som er interessant at se nærmere på. Start med at læse https://code.visualstudio.com/docs/languages/csharp.

    Snippets

    De samme snippets man kender fra Visual Studio er også tilgængelige i Visual Studio Code - eksempelvis:

    for (løkke)
if (betingelse)
class (klasse)

    og så videre.

    Extensions

    Følgende extensions har jeg installeret

    • C# (ms-dotnettools.csharp)
      • Nødvendig for C# udvikling
    • Code Runner (formulahendry.code-runner)
      • Afvikling af kode (herunder C#) direkte i VSC
