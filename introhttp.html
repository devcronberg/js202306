    Introduktion til HTTP

    HTTP er en forkortelse for (Hypertext Transfer Protocol) og er en Request/Response kommunikationsprotokol skabt af Tim Berners-Lee - se eventuelt http://ithistorie.cronberg.dk.

    Klient / Server

    Den oprindelige HTTP 1/1.1 er en relativ simpel tekstbaseret protokol hvor klienten foretage et request til serveren og server behandler det og returnerer et *response. Det kaldes også for en HTTP transaktion. Typisk sker kommunikation over port 80 ovenpå TCP/IP.

    Stuktur

    Grundlæggende består en request/response af

    • Header
    • Data
    • Body
    • FormData, HTML, CSS, JavaScript, JSON, XML

    Kommandoer

    Der findes en del forskellige kommandoer men i standard webudvikling skal du kende til GET og POST. GET kommandoen bruges til at hente data fra serveren gennem en URL, og POST bruges til at sende data til serveren via en formular eller JavaScript.

    Status koder

    Der findes en del status koder som du som udvikler kan bruge til at finde status på kommunikationen:

    • Informational 1XX
    • Successful 2XX
    • 200 OK
    • Redirection 3XX
    • 301/302
    • Client Error 4XX
    • Server Error 5X
    • 500 Server error

    Cookie

    En Cookie er i virkeligheden bare et lille stykke tekst som inkluderes i headeren. Afhængig af typen vil browseren opbevare teksten og medtage denne til serveren ved alle efterfølgende kald indtil den udløber.

    Her er et eksempel på en server som sender en cookie:

    HTTP/1.1 200 OK
Cache-Control: private
Content-Length: 1765
Content-Type: text/html; charset=utf-8

    Kryptering

    HTTP er den rene tekstbaserede protokol. De fleste websites benytter en SSL krytering ved hjælp af et certifikat og komunikere over en anden port end port 80. I så fald kaldes protokollen HTTPS.

    HTTP/2

    For ikke så mange år siden blev HTTP 1.1 opdateret til HTTP 2 og alle browsere og servere har i dag mulighed for at kommunikere over HTTP2 hvis det ønskes. HTTP2 er en kraftig opdatering af den gamle version med en masse ny features:

    • Multiplexing (flere transaktioner i en)
    • Server push
    • Binær
    • Prioritisering af afsendelse af ressourcer
    • Hastighed

    Se eventuelt HTTP2 test

    URI/URL

    Al kommunikation sker ved hjælp af en tekst som identificere en ressource på nettet. Oprindelig specificerede Tim Berners-Lee URI (Uniform Resource Identifier) men URL (Uniform Resource Locator) og URN (Uniform Resource Name) er de konkrete specifikationer benyttet i HTTP.

    En URI består af 

    Protokol://subdomæne(r).domæne/resource?Querystring (opdelt i nøgle og værdier adskilt med &)

    Her er et par eksempler:

    http://www.google.com
https://kursusportal.cronberg.dk/forside
https://kursusportal.cronberg.dk/modul/1
https://kursusportal.cronberg.dk/søg?q=web
https://kursusportal.cronberg.dk/viskursus?id=1&bruger=michell

    Værktøj

    Der findes en del værktøjer relateret til test og brug af HTTP.

    F12

    Som udvikler kan du typisk nøjes med at benytte netværksfanen i browseren F12 tool - den giver alle de oplysninger der ønskes.

    Fiddler

    Men hvis du gerne vil have en dybere forståelse for HTTP og samtidigt gerne vil kunne skabe scripts der tester HTTP kommunikation kan Fiddler anbefales.

    Curl

    Slutteligt findes Curl som er et meget benyttet kommandolinje værktøj til alle former for HTTP kommunikation.
