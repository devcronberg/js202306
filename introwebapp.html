    Introduktion til web apps

    Jo mere der kan afvikles på klienten (typisk browseren) jo bedre. Det vil spare ressourcer på serveren og gøre brugeroplevelsen bedre. Det var især biblioteket jQuery som kickstartede klientbaseret udvikling fordi det blev nemt og sikkert (i forhold til gamle browser versioner) at opdatere DOM, generere brugerflade elementer og foretage HTTP API kald.

    Hvis man ikke har behov for en server (måske bortset fra at levere statisk HTML, CSS og JavaScript) kan man sagtens benytte almindelig JavaScript eller biblioteker som jQuery til at skabe en applikation. Her er eksempelvis en side som beregner arealet på en firkant i almindelig JavaScript.

    Eksempel på en “rå” JS applikation

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Areal</title>
    <style>
      .input span {
        display: inline-block;
        width: 100px;
        margin: 10px;
      }
      button {
        margin: 10px;
      }
    </style>
  </head>
  <body>
    <h1>Beregning af areal</h1>
    <div class="input">
      <div><span>Længde</span><input id="længde" type="number" /></div>
      <div><span>Bredde</span><input id="bredde" type="number" /></div>
      <div><span>Areal</span><input id="areal" type="number" readonly /></div>
      <div><button id="beregn">Beregn</button></div>
    </div>
    <script>
      document.querySelector("#beregn").onclick = () => {
        let l = parseFloat(document.querySelector("#længde").value);
        let b = parseFloat(document.querySelector("#bredde").value);
        let a = l * b;
        document.querySelector("#areal").value = a;
      };
    </script>
  </body>
</html>

    Eksempel på en “rå” SPA applikation

    I moderne applikationer vil man gerne holde brugeren på samme side for at gøre oplevelsen bedre og bevare tilstand. Prøv følgende eksempel:

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Areal</title>
    <style>
      .input span {
        display: inline-block;
        width: 100px;
        margin: 10px;
      }
      button {
        margin: 10px;
      }
    </style>
  </head>
  <body>
    <h1>Beregning af areal</h1>
    <div>[<a href="#" id="menuFirkant">Firkant</a>] [<a href="#" id="menuTrekant">Trekant</a>]</div>

    <section id="firkant">
      <h2>Firkant</h2>
      <div class="input">
        <div><span>Længde</span><input id="længde" type="number" /></div>
        <div><span>Bredde</span><input id="bredde" type="number" /></div>
        <div><span>Areal</span><input id="arealf" type="number" readonly /></div>
        <div><button id="beregnf">Beregn</button></div>
      </div>
    </section>
    <section id="trekant">
      <h2>Trekant</h2>
      <div class="input">
        <div><span>Højde</span><input id="højde" type="number" /></div>
        <div><span>Grundlinje</span><input id="grundlinje" type="number" /></div>
        <div><span>Areal</span><input id="arealt" type="number" readonly /></div>
        <div><button id="beregnt">Beregn</button></div>
      </div>
    </section>

    <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
    <script>
      // Når siden loader vises firkant
      $("#trekant").hide();
      $("input").val(0);

      $("#menuFirkant").click(() => {
        $("#trekant").hide();
        $("#firkant").show();
        $("input").val(0);
      });
      $("#menuTrekant").click(() => {
        $("#trekant").show();
        $("#firkant").hide();
        $("input").val(0);
      });

      $("#beregnf").click(() => {
        let l = parseFloat($("#længde").val());
        let b = parseFloat($("#bredde").val());
        let a = l * b;
        $("#arealf").val(a);
      });
      $("#beregnt").click(() => {
        let h = parseFloat($("#højde").val());
        let g = parseFloat($("#grundlinje").val());
        let a = h * g * 0.5;
        $("#arealt").val(a);
      });
    </script>
  </body>
</html>

    Det bliver dog hurtigt for komplekst selv at styre tilstand, links udefra (rute), sikkerhed, DOM manipulation med videre. Derfor findes de efterhånden mange SPA frameworks.

    Eksempel på HTTP kald (AJAX)

    Dette eksempel benytter både jQuery og underscore

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>Kommuner i DK</title>
  </head>
  <body>
    <h1>Kommuner i Danmark</h1>

    <button>Hent info om alle kommuner</button>

    <ul id="lst" />
    <script src="https://cdnjs.cloudflare.com/ajax/libs/underscore.js/1.11.0/underscore-min.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
    <script>
      $("button").click(() => {
        const ul = $("#lst");
        ul.empty();
        let data = $.get("https://dawa.aws.dk/kommuner/", (data) => {
          // skab et array af objekter med navn og kode (skærer resten fra)
          let navne = _.map(data, (o) => ({
            navn: o.navn,
            kode: o.kode,
          }));
          navne.forEach((v) => {
            let li = $("<li/>").html(`${v.kode}: ${v.navn}`);
            ul.append(li);
          });
        });
      });
    </script>
  </body>
</html>

    SPA Applikationer

    SPA er en forkortelse for Single Page Application. og man kan komme langt med udvikle egne simple “enkelt side” applikationer med rå JavaScript eller eksempelvis jQuery. Men man løber hurtigt panden mod muren når applikationen vokser og kommer til at mangle hjælp til

    • komponentbaseret udvikling
    • databinding
    • links til funktioner - både interne og eksterne
    • tilstand
    • sikkerhed
    • generelle udviklingsværktøjer til fejlfinding mv

    Derfor findes der en del SPA frameworks der kan hjælpe.

    Forskellige frameworks

    Der er især 5 framework som bliver brugt meget

    • Angular (Google)
    • React (Facebook)
    • Vue
    • Ember
    • Svelte

    De er alle forskellige fra hinanden og det er lidt en religion hvilket framework man foretrækker. Angular er nok mest brugt men er sammen med React nok også de sværeste at lære. Vue er noget nemmere at komme igang med.

    SPA udviklingsprocessen

    En af de ting som gør udviklingsprocessen kompleks i relation til SPA er applikationer er, at den komponentbaserede udvikling kræver brug af pakkemoduler som WebPack, Browserify eller Parcel. Så det er typisk en relativ kompleks Node-applikation der skal skabes for at understøtte en SPA udvikling.

    CLI

    De fleste framework kommer med en CLI (Command Line Interface) som kan bruges til at

    • Oprette applikationer
    • Vedligeholde applikationer
    • Afvikle applikationer
    • Bygge applikationer
